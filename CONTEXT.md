# moka-1 Context — アプリ

moka-1(エージェント型RSSフィードリーダー)のアプリコンテキスト(moka-core / moka-web / db 共通)のドメイン用語集。実装詳細は書かない。他コンテキストは [CONTEXT-MAP.md](./CONTEXT-MAP.md) 参照。

## Language

### 原則・経路

**One-kick**:
`git clone && docker compose up -d` で読める状態まで完結する立ち上げ原則(tenets §2-1)。マイグレーション・モデル取得・初期設定は起動シーケンス内で自動化される。
_Avoid_: ワンコマンド起動、セットアップレス

**コアパス**:
LLM 無しで成立する経路 — フィード取得→正規化→DB書き込み→「読める」まで。フェイルソフトの境界線(tenets §3.2)。
_Avoid_: クリティカルパス、基本機能(境界が曖昧になる)

**増強 (enrichment 系機能の総称)**:
コアパスの外側にある「あれば嬉しい」機能群 — 濃縮・ハイライト・問い返し・取り寄せ。死んでも素のRSSリーダーとして動き続ける。
_Avoid_: 拡張機能(プラグインの含意)、AI機能(取り寄せは AI でない)

**フェイルソフト**:
アプリ内の障害方針 — 増強が失敗してもコアパスは止めない(tenets §2-6)。エッジの Fail-closed とは逆方向なので所属を必ず明示する。
_Avoid_: グレースフルデグラデーション(カタカナ長語)、fail-closed(エッジの語)

### エージェントループ

**常駐エージェント**:
moka-core 内で「取得→濃縮→提示」を自律的に回す単一の常駐ループ(tenets §2-2)。cron の寄せ集めではない。UI 人格の「給仕」とは区別する。
_Avoid_: ワーカー(one-shot ジョブと紛れる)、バッチ、デーモン

**濃縮 (enrichment)**:
LLM による要約・タグ・埋め込みの生成(tenets §3.2)。AI 生成物として常に fujinezu 面に載る。取り寄せを含まない。
_Avoid_: エンリッチ、後処理、加工

**取り寄せ**:
読者の求めに応じて外部サイトから全文を連れてくる行為。濃縮とは別概念 — 濃縮は moka が淹れるもの(fujinezu 面)、取り寄せは記事の声そのもの(kumoi 地)。ボタン文言は「全文を取り寄せる」。
_Avoid_: フェッチ、クロール、スクレイピング

**ハイライト**:
提案層3系統の推薦を集約層が理由付きで統合した「今日読むべき記事」(MoA のミニ適用、tenets §3.4)。
_Avoid_: おすすめ、レコメンド(協調フィルタリング的な含意を避ける)

**問い返し**:
記事を開いた状態での質問に、購読履歴 RAG + 当該記事コンテキストで答える機能(tenets §3.4)。UI 上の入口は「訊く」入力バー。
_Avoid_: チャット、Q&Aセッション(セッション永続の含意)

**ハイブリッド検索**:
テキスト一致とベクトル近傍を融合して記事を探す検索方式(tenets §2-4)。問い返しの検索層と、ホームの検索UIの両方を支える。
_Avoid_: セマンティック検索(ベクトル片翼だけを指す)、全文検索(FTSの含意が強く日本語で誤解を生む)

**リキャップ**:
購読記事の週次まとめ。交代枠モデルによる品質最優先のバッチ生成。
_Avoid_: ダイジェスト、週報

### LLM編成

**高速パス(LLMの役割)**:
常駐 LLM 編成のうち、要約・タグ抽出・翻訳を担う軽量モデルの役割名(現行 Qwen3.5-4B、ADR00007)。
_Avoid_: fast path(Plecto のネイティブ Rust 側と衝突する。エッジの話なら「Plecto の fast path」と限定する)

**提案層**:
ハイライト候補を3系統で独立に推薦する常駐モデル群の役割名(tenets §3.3)。
_Avoid_: 推薦モデル、候補生成器

**集約層**:
提案層の出力を審査・統合し、RAG Q&A の最終回答も担う大型常駐モデルの役割名(現行 gpt-oss-20B)。
_Avoid_: 審判(eval コンテキストの語)、オーケストレータ

**交代枠 (swap slot)**:
常駐せず、品質最優先のバッチ(週次リキャップ等)の時だけロードされるモデル枠(現行 Qwen3.6-35B-A3B、ADR00004)。
_Avoid_: スワップ枠(表記揺れ)、オンデマンドモデル

### 記事と本文

**全文**:
記事のリンク先から取り寄せた本文全体。RSS 由来の content(概要のことが多い)と区別する。記事の声なので明朝・kumoi 地に載る(AI 生成物ではない)。
_Avoid_: フルテキスト、本文抽出結果(工程の語であって呼び名ではない)、原文(リンク先サイトそのもの・対訳の原語側と衝突する)

### Web UI

**ホーム**:
アプリの入口 `/`。サイドバー(記事リスト)と読書カラムの2ペイン構成で、記事未選択時の読書カラムは招待の空状態を示す。
_Avoid_: ランディングページ(マーケティング LP の含意)、トップページ、ダッシュボード

**読書ビュー**:
記事1本を読むための画面。本文(記事の声 = 明朝)を主役に、AI 生成物(要約・対訳・Q&A)は fujinezu 面の給仕として従属する。
_Avoid_: 記事詳細、詳細ページ(読むための場所であることが消える)

**読書カラム**:
2ペイン構成の右側、本文が流れる最大38字測度のカラム。読書ビューの構成要素。
_Avoid_: メインエリア、コンテンツ領域

**フィード管理**:
フィードの登録・削除と登録済み一覧の画面(`/feeds`)。ホームの空状態にも登録導線が現れるが、常設の場所はここ。
_Avoid_: 設定ページ(将来の設定項目一般と混同する)、フィード設定

**フィードの削除**:
店との別れ。フィードを購読から外し、その店の記事・濃縮の成果・既読の事実ごと手放す操作。取り消せないため必ず確認を挟む。
_Avoid_: 購読解除(記事が残る含意)、アンサブスクライブ

**既読**:
読書ビューで記事を開いた事実。未読とは「既読の事実が無い」状態であり、リストでは濃淡で静かに沈めるのみ — 数を数えて表示しない(急かさない店)。
_Avoid_: 開封、未読数・未読バッジ(急かす含意)

**対訳**:
原文と訳文を段落単位で対にして読む表示形。デスクトップは左右2カラム、モバイルは段落交互に降格する。訳文は AI 生成物なので fujinezu 面に載る。
_Avoid_: 翻訳ビュー(表示形でなく機能を指してしまう)、パラレルテキスト

**ドリップインジケータ**:
待ち全般(推論・取り寄せ)を示す moka-1 唯一の装飾アニメーション(ruri の滴と液面)。回転スピナーの代替であり、常に工程コピーと併記する。金の表情(金の一滴)は推論完了の瞬間に限る。
_Avoid_: スピナー、ローダー、ローディングアイコン

**工程コピー**:
待ちの内部状態を喫茶の工程語に写した表示文言(「豆を挽いています」「蒸らしています」「取り寄せています」等)。技術用語を混ぜない。
_Avoid_: ローディングメッセージ、ステータステキスト

**給仕**:
UI における moka(AI)の人格規定 — 静かで丁寧、記事より前に出ない。デザイン原則「記事が主役、AI は給仕」の主語。
_Avoid_: アシスタント、エージェント(常駐エージェントループと衝突する)
