# plecto/ Context — エッジ

moka-1 のエッジコンテキストのドメイン用語集。Plecto 本体([Plecto](https://github.com/Kaikei-e/Plecto))のドメイン語に従属し、moka-1 固有の運用語だけを載せる。実装詳細は書かない。他コンテキストは [CONTEXT-MAP.md](../CONTEXT-MAP.md) 参照。

## Language

**ドッグフーディング**:
Plecto を moka-1 の実トラフィックで試験運用すること(tenets §1.2)。踏んだ問題は回避せずPlecto側にissue/ADR起票するのが本義。
_Avoid_: 検証、テスト運用(「実戦投入」の含意が消える)

**素通し**:
フィルタチェーンが空でTLS終端とルーティングのみ行う現在地(Phase 1)。認証・レート制限フィルタが乗る前の状態。
_Avoid_: パススルー、プロキシのみ

**Phase(1〜4)**:
ドッグフーディング計画の段階(tenets §3.5)。1=素通し、2=WASMフィルタ第1弾(認証・レート制限)、3=on-request-bodyフィルタ、4=OCI配布+署名+ホットリロード運用試験。
_Avoid_: マイルストーン(アプリ側のM0〜M5と紛れる)

**Fail-closed**:
エッジの障害方針 — 壊れたmanifest編集やフィルタ異常は無視・遮断する側に倒す。アプリ内のフェイルソフトとは逆方向。
_Avoid_: フェイルセーフ(意味が曖昧)、フェイルソフト(アプリ側の語と衝突)

**セッション認証フィルタ**:
Phase 2 第1弾のWASMフィルタ。リクエストのセッション資格(署名cookie)を検証し、無効なら遮断する(Fail-closed)。認証の儀式(パスキー登録・ログイン)はアプリ側の仕事で、フィルタは検証だけを担う。
_Avoid_: 認証フィルタ(儀式まで担う含意)、ログインフィルタ

**レート制限フィルタ**:
Phase 2 第1弾のWASMフィルタ。host-ratelimit で流量を絞る。全経路の per-IP 制限と、ログイン経路の厳しめ制限の二段構え。
_Avoid_: スロットリング(縮退運転の含意と紛れる)

**upstream 契約**:
Plecto配下の各サービスが満たすべき規約 — `GET /healthz`に200を返すまでルーティングに入らない、平文HTTP/1.1で受ける、`/api` prefixはstripされない。
_Avoid_: ヘルスチェック規約(規約全体でなく一部を指してしまう)
