# CONTEXT.md フォーマット

moka-1 の用語集(グロッサリー)の書式。正準ソースは `docs/tenets/moka-tenets.md` と `docs/adr/` — CONTEXT.md の用語がこれらと矛盾してはならない。

## 構造

```md
# moka-1 Context

moka-1(エージェント型RSSフィードリーダー)のドメイン用語集。実装詳細は書かない。

## Language

**濃縮 (enrichment)**:
取得済み記事への AI 増強 — 要約・タグ抽出・埋め込み生成。`enrichment_status` で冪等に再処理される。
_Avoid_: 加工、後処理、エンリッチメント処理(カタカナ長音表記の揺れ)

**コアパス**:
LLM 無しで成立する経路 — フィード取得→正規化→DB書き込み→「読める」まで。フェイルソフトの境界線。
_Avoid_: クリティカルパス、基本機能(境界が曖昧になる)

**高速パス(LLMの役割)**:
常駐 LLM 編成のうち、要約・タグ抽出・翻訳を担う軽量モデルの役割名(現行 Qwen3.5-4B)。
_Avoid_: fast path(Plecto のネイティブ Rust 側と衝突する。エッジの話なら「Plecto の fast path」と限定する)

**ハイライト**:
提案層3系統の推薦を集約層が理由付きで統合した「今日読むべき記事」。MoA のミニ適用。
_Avoid_: おすすめ、レコメンド(協調フィルタリング的な含意を避ける)
```

## ルール

- **意見を持つ。** 同じ概念に複数の言い方があるなら最良の一語を選び、残りは `_Avoid_` に列挙する。
- **定義は締める。** 1〜2文まで。それが「何であるか」を定義し、動作の説明はしない。
- **moka-1 固有の概念だけを載せる。** 一般的なプログラミング概念(タイムアウト、エラー型、ユーティリティパターン)は、プロジェクトで多用されていても載せない。追加前に自問する: これはこのコンテキスト固有の概念か、一般概念か。前者だけが載る。
- **表記の正準は tenets/ADR に合わせる。** 日本語用語には初出時に英語対訳を括弧で併記してよい(例: 濃縮 (enrichment))。コード上の識別子(`enrichment_status` 等)と対応する用語は、その対応を定義文に一言含める。
- **Plecto 側の用語と衝突する語は必ず限定修飾する。** filter・fast path・decision 等は Plecto のドメイン語。moka-1 の文脈で使うときは「Plecto の〜」「WASM フィルタ」のように所属を明示する。
- **自然なまとまりが見えたら小見出しで分ける**(例: エージェントループ / LLM編成 / エッジ)。全体が一塊なら平坦なリストでよい。

## 単一コンテキスト vs 複数コンテキスト

**単一コンテキスト(moka-1 の現在):** リポジトリルートに `CONTEXT.md` を1枚。moka-1 は「単一言語・単一DB・少数コンテナ」を掲げるミニマル構成であり、用語集も1枚に収まるのが健全。分割したくなったら、それ自体をミニマリズム逸脱のシグナルとして疑う。

**複数コンテキスト(将来、境界が本当に分かれたら):** ルートに `CONTEXT-MAP.md` を置き、各コンテキストの所在と関係を列挙する:

```md
# Context Map

## Contexts

- [アプリ](./CONTEXT.md) — エージェントループ、濃縮、ハイライト、RAG(moka-core / moka-web 共通のドメイン語)
- [エッジ](./plecto/CONTEXT.md) — WASM フィルタ、認証、レート制限(Plecto のドメイン語に従属)
- [評価](./eval/CONTEXT.md) — モデル A/B、プロンプト評価の用語

## Relationships

- **エッジ → アプリ**: Plecto はすべての外部トラフィックを終端し、フィルタチェーンを通してから moka-web / moka-core に渡す
- **評価 → アプリ**: eval/ の A/B 結果が LLM 編成(高速パス・提案層・集約層)のモデル選定を駆動する(結果は ADR に記録)
```

スキルはどちらの構造かを推論する:

- `CONTEXT-MAP.md` があれば、それを読んでコンテキストを特定する
- ルートの `CONTEXT.md` だけなら単一コンテキスト
- どちらも無ければ、最初の用語が確定した時点でルートに `CONTEXT.md` を遅延作成する

複数コンテキストがある場合、現在の話題がどれに属するかを推論する。不明なら訊く。

## 用語集に載せないもの

- 実装の決定(モデル選定、ライブラリ採否、構成変更)→ `docs/adr/`(ADR00001.md からの連番、`moka-adr-writer` スキル)
- 設計原理と背景 → `docs/tenets/moka-tenets.md`
- スキーマやAPIの詳細 → `db/schema.sql`、コード自身
