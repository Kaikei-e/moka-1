# eval/ Context — 評価

moka-1 のモデル選定・プロンプト評価コンテキストのドメイン用語集。実装詳細は書かない。他コンテキストは [CONTEXT-MAP.md](../CONTEXT-MAP.md) 参照。

## Language

**Phase R 調査**:
A/B実験を組む前に方法論(審判プロトコル・データセット設計)を出典URL付きで固める調査段階。結果は`results/research/`に置き、結論は`docs/adr/`へ還流する。
_Avoid_: 事前調査、リサーチフェーズ(アプリ側のマイルストーンと紛れる)

**主審**:
候補モデルと別族の最強単一審判モデル(現行gpt-oss-20b)。弱い異種審判の多数決は組まない設計判断。
_Avoid_: メインジャッジ、一次審判

**追加審判**:
主審と独立に判定を行うClaude審判。ローカル審判(主審)との一致率自体がローカル審判のメタ評価になる。
_Avoid_: セカンダリジャッジ、副審(役割の非対称性が消える)

**同族バイアス**:
審判モデルが自分自身や近い系統のモデルの出力を過大評価する傾向。gpt-oss自身が関わる比較(recap等)では主審をClaudeに切り替え、gpt-oss判定は参考値に格下げする。
_Avoid_: セルフバイアス、自己選好

**位置スワップ**:
ペア比較でA/Bの提示順を入れ替えて2回判定させる手法。両順で同じ勝者に決着した場合のみ「勝ち」として採用する。
_Avoid_: 順序バイアス補正(手法名でなく目的を指してしまう)

**GPUシリアル(計測)**:
同一iGPUの取り合いを避けるため、評価計測中は`llm`サービスを停止し計測完了後に再開する運用規約。
_Avoid_: 排他制御(コード用語が先に立つ)、シリアライズ
